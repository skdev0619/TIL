# 빈 스코프

### 빈 스코프 종류(싱글톤, 프로토타입, 웹 관련)?
- 싱글톤 : 스프링 컨테이너 시작과 종료
- 프로토타입 : 스프링 컨테이너에서 빈 생성, 의존관계 주입, 초기화까지만 담당
- 웹 관련
    - request: 웹 요청이 들어오고 나갈때 까지 유지
    - session: 웹 세션이 생성되고 종료될 때 까지 유지
    - application: 웹의 서블릿 컨텍스트와 같은 범위로 유지

### 스코프 등록 방법?
> @Scope("prototype")

### 프로토타입 스코프란?
- 스프링 컨테이너에서 조회 시 항상 새로운 빈만 리턴
- 스프링 컨테이너에서 빈 생성, 의존 관계 주입, 초기화만 담당하고 빈 관리는 하지 않는다.
- 그래서 @PreDestroy 같은 종료 메서드 호출되지 않는다. 그래서 종료 메서드는 클라이언트가 직접 해야 한다.

### 프로토타입과 싱글톤빈 같이 사용해서 문제되는 케이스?
- 싱글톤이 프로토타입 빈을 주입받아서 사용하는 케이스
- `싱글톤 빈이 내부에 가지고 있는 프로토타입빈은 사용할 때마다 새로 생성되지 않는다` 
- 프로토타입 빈이 생성되기는 하지만 싱글톤 빈은 생성시점에만 의존관계 주입받아서 싱글톤 빈과 함께 유지되는 문제

### 프로토타입과 싱글톤빈 같이 사용 시 문제 해결방법?
